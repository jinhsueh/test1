# PRD：健康保健品牌內容行銷自動化系統

## 文件資訊
- **版本**：v1.0
- **日期**：2026-03-30
- **狀態**：草稿

---

## 1. 背景與問題

### 1.1 現況痛點
- 健康保健產品在品牌官網直接投放廣告，受到**健康食品廣告法規**嚴格限制
- 目前缺乏穩定的自然流量來源，過度依賴付費廣告
- 社群內容產出缺乏系統化流程，人力成本高且產出不穩定

### 1.2 策略方向
建立一個**衛教內容網站**作為中間層，透過高品質內容吸引自然流量，再間接導流至品牌官網與電商網站，合規地完成銷售轉換。

---

## 2. 目標

| 目標 | 衡量指標 |
|------|---------|
| 建立穩定的內容產出機制 | 每日至少產出 1 篇圖文內容 |
| 提升自然流量 | 月流量逐月成長 20% |
| 有效導流至官網/電商 | 內容網站 → 官網的點擊轉換率 ≥ 3% |
| 合規經營 | 零法規違規事件 |

---

## 3. 系統架構

### 3.1 導流漏斗

```
┌─────────────┐    ┌──────────────────┐    ┌─────────────────┐
│  社群平台     │ →  │  衛教內容網站      │ →  │ 品牌官網/電商網站  │
│  (IG/FB等)   │    │  (文章/圖文)       │    │  (銷售轉換)       │
└─────────────┘    └──────────────────┘    └─────────────────┘
     曝光               信任建立                 轉換
```

### 3.2 雙網站定位

| 網站 | 定位 | 內容調性 | 法規風險 |
|------|------|---------|---------|
| **衛教內容網站** | 健康知識平台 | 客觀、教育性、知識分享 | 低（衛教不受廣告法規限制） |
| **品牌官網/電商** | 產品銷售 | 品牌調性、產品導向 | 高（需嚴格遵守廣告法規） |

## 4. 功能需求

### 4.1 內容來源模組

#### A. 外部資訊收集引擎
- **功能**：每日自動收集產業相關新聞、熱門話題、趨勢資訊
- **規則**：
  - 篩選與自家產品可關聯的話題
  - 識別高流量/高搜尋量的關鍵字
  - 標記內容分類（見 4.2）
- **輸出**：每日產出「今日話題清單」，供內容生成模組使用

#### B. 內部產品資料庫
- **功能**：維護產品資訊庫，供內容生成時引用
- **資料欄位**：
  - 產品名稱、特色、成分
  - 適用情境、使用方式
  - 可宣稱的功效（已通過法規審核）
  - 相關衛教知識關鍵字
- **規則**：僅引用已審核通過的產品宣稱

### 4.2 內容分類體系

| 分類 | 說明 | 範例 | 導購強度 |
|------|------|------|---------|
| **衛教類** | 健康觀念、疾病預防、保健知識 | 「腸道健康與免疫力的關係」 | ★☆☆☆☆ |
| **教育類** | 成分科普、營養學知識 | 「益生菌的5大菌株比較」 | ★★☆☆☆ |
| **趨勢類** | 產業趨勢、最新研究 | 「2026年保健食品三大趨勢」 | ★★☆☆☆ |
| **生活類** | 日常保健tips、食譜、養生法 | 「上班族的5分鐘晨間保健法」 | ★★★☆☆ |
| **季節類** | 換季保養、節氣養生 | 「春天過敏季的3個保養關鍵」 | ★★★☆☆ |
| **QA類** | 常見問題解答、迷思破解 | 「益生菌飯前吃還是飯後吃？」 | ★★★★☆ |
| **見證類** | 使用心得、案例分享 | 「營養師的一日保健菜單」 | ★★★★☆ |
| **活動類** | 促銷活動、品牌活動、節慶 | 「母親節健康禮盒推薦」 | ★★★★★ |

### 4.3 內容自動生成模組

- **輸入**：外部話題 + 內部產品資料 + 指定分類
- **輸出**：
  - 圖文內容（適用 IG/FB 發文格式）
  - 文章內容（適用內容網站 SEO 文章）
- **規則**：
  - 衛教類內容不得出現直接產品推銷語句
  - 所有健康宣稱需有參考來源
  - 導購 CTA 以「了解更多」「延伸閱讀」等軟性方式呈現
  - 自動嵌入可追蹤的導流連結

### 4.4 多平台發布與排程模組

#### 支援平台與內容適配

| 平台 | 內容格式 | 特性 | 導流方式 | 串接方式 |
|------|---------|------|---------|---------|
| **Instagram** | 圖文貼文、Reels、限時動態、Carousel | 視覺導向，年輕族群 | Bio 連結 / 限動連結貼紙 | Meta Graph API |
| **Facebook** | 圖文貼文、影片、連結貼文 | 可直接放連結，觸及廣泛 | 貼文內嵌連結 | Meta Graph API |
| **LINE 官方帳號** | 圖文訊息、Rich Menu、推播 | 高開信率，適合忠實客群 | 圖文訊息按鈕連結 | LINE Messaging API |
| **Blog / 內容網站** | SEO 長文、專題文章 | 長尾流量，Google 搜尋 | 文內 CTA 連結 | CMS API (WordPress等) |
| **Threads** | 短文、圖文串 | 文字互動導向 | 貼文內嵌連結 | Meta Threads API |
| **X (Twitter)** | 短文、圖文 | 話題擴散快 | 貼文內嵌連結 | X API v2 |
| **YouTube** | 短影音 (Shorts)、長影片 | 影片搜尋長尾流量 | 資訊欄連結 / 置頂留言 | YouTube Data API |
| **TikTok** | 短影音 | 年輕族群、高曝光潛力 | Bio 連結 | TikTok API |

#### 內容自動適配引擎

同一篇內容原稿自動轉換為各平台最適格式：

```
                    ┌→ IG 圖文 (1080x1080 視覺圖 + 短文案 + hashtag)
                    ├→ IG Reels (9:16 短影音 + 字幕)
                    ├→ FB 連結貼文 (預覽圖 + 摘要 + 連結)
   內容原稿 ────────├→ LINE 圖文訊息 (圖片 + 按鈕 CTA)
   (文章+圖片)      ├→ Blog SEO 長文 (完整文章 + 結構化標記)
                    ├→ Threads 短文串 (拆分為多則短文)
                    ├→ X 短文 (280字摘要 + 連結)
                    └→ YouTube/TikTok 短影音腳本
```

**適配規則**：
- **文案長度**：IG ≤ 2200 字、X ≤ 280 字、LINE 推播 ≤ 500 字、Blog ≥ 800 字
- **圖片尺寸**：自動裁切為各平台最佳比例（1:1、9:16、16:9、1.91:1）
- **Hashtag 策略**：IG 最多 30 個（建議 10-15）、X 最多 3 個、FB 不加
- **語氣調性**：IG 輕鬆活潑、Blog 專業深入、LINE 親切直接

#### 發布排程策略

| 平台 | 建議發文頻率 | 最佳時段 | 優先分類 |
|------|------------|---------|---------|
| **Instagram** | 每日 1-2 則 | 12:00-13:00 / 20:00-21:00 | 生活、衛教、見證 |
| **Facebook** | 每日 1 則 | 11:00-12:00 / 19:00-20:00 | 教育、趨勢、活動 |
| **LINE** | 每週 2-3 則 | 10:00-11:00 | 活動、QA、季節 |
| **Blog** | 每週 2-3 篇 | N/A（SEO 導向） | 衛教、教育、趨勢 |
| **Threads** | 每日 1-2 則 | 08:00-09:00 / 21:00-22:00 | QA、趨勢 |
| **X** | 每日 1-3 則 | 08:00-09:00 / 12:00-13:00 | 趨勢、QA |

- 依分類比例自動安排內容組合（例：衛教 30%、教育 20%、生活 15%、趨勢 10%、季節 10%、QA 5%、見證 5%、活動 5%）
- 系統自動偵測各平台最佳發文時段，動態調整排程
- 支援手動介入排程調整與內容審核後再發布

#### 導流追蹤連結

- 每篇內容自動產生 UTM 追蹤連結
- UTM 參數包含：來源平台、內容分類、日期
- 各平台 UTM 範例：
  - IG：`?utm_source=instagram&utm_medium=post&utm_campaign=education&utm_content=20260330`
  - FB：`?utm_source=facebook&utm_medium=post&utm_campaign=health_edu&utm_content=20260330`
  - LINE：`?utm_source=line&utm_medium=push&utm_campaign=promotion&utm_content=20260330`
  - Blog：`?utm_source=blog&utm_medium=article&utm_campaign=seo&utm_content=20260330`
- 短網址服務整合（如 Bitly / 自建短網址），追蹤點擊數據
- 各平台導流連結獨立追蹤，儀表板可按平台篩選成效

### 4.5 成效儀表板

#### 每日報表
- 各平台發文數 / 曝光數 / 互動數
- 內容網站總流量與來源分布
- 內容網站 → 官網/電商的導流點擊數
- 各分類文章的表現排名

#### 週報/月報
- 流量趨勢圖（按日/按週）
- 各內容分類的成效比較
- 導流轉換漏斗分析
- 最佳表現文章 TOP 10
- 自動產出下週內容策略建議

#### 即時看板
- 今日發文狀態
- 今日即時流量
- 導流連結點擊即時統計

### 4.6 自動優化模組

- 根據歷史文章成效，自動調整：
  - 各分類的內容配比
  - 發文時段
  - 標題與 CTA 風格
- 每日自動分析前一日內容表現，產出優化建議

## 5. 法規合規設計

### 5.1 內容防火牆

```
衛教內容網站                    品牌官網/電商
┌─────────────────┐           ┌──────────────────┐
│ ✅ 客觀衛教知識    │           │ ✅ 產品資訊        │
│ ✅ 研究引用       │    導流    │ ✅ 經核准的廣告用語  │
│ ✅ 生活保健建議    │ ────────→ │ ✅ 合規的促銷活動   │
│ ❌ 產品直接推銷    │           │                   │
│ ❌ 療效宣稱       │           │                   │
└─────────────────┘           └──────────────────┘
```

### 5.2 自動審核規則
- 內容生成後自動掃描違規關鍵字（療效宣稱、誇大用語）
- 衛教文章中不得出現產品名稱的直接推銷
- 導流 CTA 使用中性用語（「延伸閱讀」「了解更多」）
- 建立違規詞庫，持續更新

## 6. 技術架構（概要）

```
┌──────────────────────────────────────────────────────────────┐
│                      排程引擎 (Scheduler)                      │
├──────────────┬──────────────┬────────────────────────────────┤
│  外部資訊收集   │  內容生成引擎   │       內容適配引擎                │
│  (Crawler)    │  (AI/LLM)    │  (格式/尺寸/文案自動轉換)         │
├──────────────┴──────────────┴────────────────────────────────┤
│                     多平台發布閘道 (Gateway)                     │
├────────┬────────┬───────┬───────┬────────┬──────┬───────────┤
│  IG    │  FB    │ LINE  │ Blog  │Threads │  X   │ YT/TikTok │
├────────┴────────┴───────┴───────┴────────┴──────┴───────────┤
│                  產品資料庫 / 內容資料庫                          │
├──────────────────────────────────────────────────────────────┤
│              分析引擎 / 儀表板 (Dashboard)                       │
│     (UTM追蹤 + Google Analytics + 各平台 Insights + 自建報表)     │
└──────────────────────────────────────────────────────────────┘
```

### 主要技術元件
- **資訊收集**：RSS / 爬蟲 / Google Trends API
- **內容生成**：LLM (Claude API) + 圖片生成 + 短影音腳本生成
- **內容適配**：自動轉換文案長度、圖片尺寸、Hashtag、語氣調性
- **社群發布**：
  - Meta Graph API（Instagram / Facebook）
  - LINE Messaging API（LINE 官方帳號）
  - WordPress REST API / CMS API（Blog 內容網站）
  - Meta Threads API（Threads）
  - X API v2（X / Twitter）
  - YouTube Data API（YouTube Shorts）
  - TikTok API（TikTok）
- **追蹤分析**：Google Analytics 4 + UTM + 各平台原生 Insights API + 自建追蹤
- **儀表板**：Web Dashboard（即時數據視覺化，支援按平台篩選）
- **合規審核**：違規詞庫比對 + LLM 審核

## 9. 風險與對策

| 風險 | 影響 | 對策 |
|------|------|------|
| AI 生成內容出現違規用語 | 法規處罰 | 多層審核機制（AI審核 + 人工抽檢） |
| 社群平台演算法變動 | 曝光下降 | 多平台分散風險 + SEO 自然流量並行 |
| 內容品質不穩定 | 品牌形象受損 | 建立內容模板 + 品牌調性規範 |
| 導流效果不如預期 | ROI 不達標 | 持續 A/B 測試 + 自動優化機制 |
