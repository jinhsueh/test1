# PRD：健康保健品牌內容行銷自動化系統

## 文件資訊
- **版本**：v1.0
- **日期**：2026-03-30
- **狀態**：草稿

---

## 1. 背景與問題

### 1.1 現況痛點
- 健康保健產品在品牌官網直接投放廣告，受到**健康食品廣告法規**嚴格限制
- 目前缺乏穩定的自然流量來源，過度依賴付費廣告
- 社群內容產出缺乏系統化流程，人力成本高且產出不穩定

### 1.2 策略方向
建立一個**衛教內容網站**作為中間層，透過高品質內容吸引自然流量，再間接導流至品牌官網與電商網站，合規地完成銷售轉換。

---

## 2. 目標

| 目標 | 衡量指標 |
|------|---------|
| 建立穩定的內容產出機制 | 每日至少產出 1 篇圖文內容 |
| 提升自然流量 | 月流量逐月成長 20% |
| 有效導流至官網/電商 | 內容網站 → 官網的點擊轉換率 ≥ 3% |
| 合規經營 | 零法規違規事件 |

---

## 3. 系統架構

### 3.1 導流漏斗

```
┌─────────────┐    ┌──────────────────┐    ┌─────────────────┐
│  社群平台     │ →  │  衛教內容網站      │ →  │ 品牌官網/電商網站  │
│  (IG/FB等)   │    │  (文章/圖文)       │    │  (銷售轉換)       │
└─────────────┘    └──────────────────┘    └─────────────────┘
     曝光               信任建立                 轉換
```

### 3.2 雙網站定位

| 網站 | 定位 | 內容調性 | 法規風險 |
|------|------|---------|---------|
| **衛教內容網站** | 健康知識平台 | 客觀、教育性、知識分享 | 低（衛教不受廣告法規限制） |
| **品牌官網/電商** | 產品銷售 | 品牌調性、產品導向 | 高（需嚴格遵守廣告法規） |

## 4. 功能需求

### 4.1 內容來源模組

#### A. 外部資訊收集引擎
- **功能**：每日自動收集產業相關新聞、熱門話題、趨勢資訊
- **規則**：
  - 篩選與自家產品可關聯的話題
  - 識別高流量/高搜尋量的關鍵字
  - 標記內容分類（見 4.2）
- **輸出**：每日產出「今日話題清單」，供內容生成模組使用

#### B. 內部產品資料庫
- **功能**：維護產品資訊庫，供內容生成時引用
- **資料欄位**：
  - 產品名稱、特色、成分
  - 適用情境、使用方式
  - 可宣稱的功效（已通過法規審核）
  - 相關衛教知識關鍵字
- **規則**：僅引用已審核通過的產品宣稱

### 4.2 內容分類體系

| 分類 | 說明 | 範例 | 導購強度 |
|------|------|------|---------|
| **衛教類** | 健康觀念、疾病預防、保健知識 | 「腸道健康與免疫力的關係」 | ★☆☆☆☆ |
| **教育類** | 成分科普、營養學知識 | 「益生菌的5大菌株比較」 | ★★☆☆☆ |
| **趨勢類** | 產業趨勢、最新研究 | 「2026年保健食品三大趨勢」 | ★★☆☆☆ |
| **生活類** | 日常保健tips、食譜、養生法 | 「上班族的5分鐘晨間保健法」 | ★★★☆☆ |
| **季節類** | 換季保養、節氣養生 | 「春天過敏季的3個保養關鍵」 | ★★★☆☆ |
| **QA類** | 常見問題解答、迷思破解 | 「益生菌飯前吃還是飯後吃？」 | ★★★★☆ |
| **見證類** | 使用心得、案例分享 | 「營養師的一日保健菜單」 | ★★★★☆ |
| **活動類** | 促銷活動、品牌活動、節慶 | 「母親節健康禮盒推薦」 | ★★★★★ |

### 4.3 內容自動生成模組

- **輸入**：外部話題 + 內部產品資料 + 指定分類
- **輸出**：
  - 圖文內容（適用 IG/FB 發文格式）
  - 文章內容（適用內容網站 SEO 文章）
- **規則**：
  - 衛教類內容不得出現直接產品推銷語句
  - 所有健康宣稱需有參考來源
  - 導購 CTA 以「了解更多」「延伸閱讀」等軟性方式呈現
  - 自動嵌入可追蹤的導流連結

### 4.4 發布與排程模組

- **支援平台**：Instagram、Facebook（未來可擴充 LINE、Blog）
- **功能**：
  - 每日自動排程發文
  - 依分類比例自動安排內容組合（例：衛教 40%、教育 20%、生活 20%、其他 20%）
  - 最佳發文時段自動偵測
- **導流連結**：
  - 每篇內容自動產生 UTM 追蹤連結
  - UTM 參數包含：來源平台、內容分類、日期
  - 格式範例：`?utm_source=instagram&utm_medium=post&utm_campaign=education&utm_content=20260330`

### 4.5 成效儀表板

#### 每日報表
- 各平台發文數 / 曝光數 / 互動數
- 內容網站總流量與來源分布
- 內容網站 → 官網/電商的導流點擊數
- 各分類文章的表現排名

#### 週報/月報
- 流量趨勢圖（按日/按週）
- 各內容分類的成效比較
- 導流轉換漏斗分析
- 最佳表現文章 TOP 10
- 自動產出下週內容策略建議

#### 即時看板
- 今日發文狀態
- 今日即時流量
- 導流連結點擊即時統計

### 4.6 自動優化模組

- 根據歷史文章成效，自動調整：
  - 各分類的內容配比
  - 發文時段
  - 標題與 CTA 風格
- 每日自動分析前一日內容表現，產出優化建議

## 5. 法規合規設計

### 5.1 內容防火牆

```
衛教內容網站                    品牌官網/電商
┌─────────────────┐           ┌──────────────────┐
│ ✅ 客觀衛教知識    │           │ ✅ 產品資訊        │
│ ✅ 研究引用       │    導流    │ ✅ 經核准的廣告用語  │
│ ✅ 生活保健建議    │ ────────→ │ ✅ 合規的促銷活動   │
│ ❌ 產品直接推銷    │           │                   │
│ ❌ 療效宣稱       │           │                   │
└─────────────────┘           └──────────────────┘
```

### 5.2 自動審核規則
- 內容生成後自動掃描違規關鍵字（療效宣稱、誇大用語）
- 衛教文章中不得出現產品名稱的直接推銷
- 導流 CTA 使用中性用語（「延伸閱讀」「了解更多」）
- 建立違規詞庫，持續更新

## 6. 技術架構（概要）

```
┌─────────────────────────────────────────────────┐
│                   排程引擎 (Scheduler)             │
├────────────┬────────────┬───────────────────────┤
│ 外部資訊收集  │ 內容生成引擎  │ 社群發布模組           │
│ (Crawler)   │ (AI/LLM)   │ (IG/FB API)          │
├────────────┴────────────┴───────────────────────┤
│              產品資料庫 / 內容資料庫                  │
├─────────────────────────────────────────────────┤
│           分析引擎 / 儀表板 (Dashboard)              │
│        (UTM追蹤 + Google Analytics + 自建報表)       │
└─────────────────────────────────────────────────┘
```

### 主要技術元件
- **資訊收集**：RSS / 爬蟲 / Google Trends API
- **內容生成**：LLM (Claude API) + 圖片生成
- **社群發布**：Meta Graph API (IG/FB)
- **追蹤分析**：Google Analytics 4 + UTM + 自建追蹤
- **儀表板**：Web Dashboard（即時數據視覺化）
- **合規審核**：違規詞庫比對 + LLM 審核

## 9. 風險與對策

| 風險 | 影響 | 對策 |
|------|------|------|
| AI 生成內容出現違規用語 | 法規處罰 | 多層審核機制（AI審核 + 人工抽檢） |
| 社群平台演算法變動 | 曝光下降 | 多平台分散風險 + SEO 自然流量並行 |
| 內容品質不穩定 | 品牌形象受損 | 建立內容模板 + 品牌調性規範 |
| 導流效果不如預期 | ROI 不達標 | 持續 A/B 測試 + 自動優化機制 |
